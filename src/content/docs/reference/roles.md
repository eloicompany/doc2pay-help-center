Doc2Pay підтримує чотири ролі з різним рівнем доступу. Нижче — детальна матриця дозволів.

## Ролі

| Роль | Опис |
|------|------|
| **Адміністратор** | Повний доступ до всього акаунту |
| **Головний бухгалтер** | Експорт, API ключі, керування платежами |
| **Бухгалтер** | Завантаження рахунків, перегляд даних |
| **Клієнт** | Перегляд лише власних даних |

\* Клієнт бачить лише платежі, прив'язані до нього.

## Деталі по ролях

### Адміністратор

Повний контроль над акаунтом: команда, підписка, інтеграції, всі дані. Рекомендується лише для власника бізнесу або довіреної особи.

### Головний бухгалтер

Може експортувати платежі, налаштовувати банківські API, керувати рахунками. Не може змінювати склад команди та тариф.

### Бухгалтер

Працює з рахунками: завантажує, перевіряє, редагує. Не має доступу до експорту та API — це зберігає контроль над виведенням коштів.

### Клієнт

Обмежений перегляд — лише ті платежі та документи, які до нього прив'язані. Зручно для ФОП, які делегують бухгалтерію сторонній команді.

:::tip
Призначайте мінімально необхідні права: якщо людині не потрібен експорт — оберіть роль «Бухгалтер».
:::
