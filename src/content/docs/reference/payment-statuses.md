Кожен платіж у Doc2Pay проходить кілька етапів. Нижче — повний опис статусів та їх зміни.

## Таблиця статусів

| Статус | Колір | Опис |
|--------|-------|------|
| **Новий** | Синій | Щойно створений платіж, ще не оброблений |
| **В обробці** | Жовтий | AI аналізує документ та заповнює дані |
| **Готовий** | Зелений | Дані перевірені, платіж готовий до відправки в банк |
| **Оплачено** | Сірий | Платіж виконано у банку |
| **Відхилено** | Червоний | Помилка при обробці або відхилення |

## Переходи між статусами

```
Новий → В обробці → Готовий → Оплачено
                ↘ Відхилено
```

### Новий → В обробці

Після завантаження файлу платіж автоматично переходить у статус «В обробці». AI розпізнає текст, витягує реквізити та заповнює форму.

### В обробці → Готовий

Якщо розпізнавання пройшло успішно, платіж стає «Готовий». Користувач може перевірити та відредагувати дані перед експортом.

### В обробці → Відхилено

Якщо AI не зміг розпізнати документ або виникла помилка — платіж відхиляється. Потрібно завантажити файл повторно або ввести дані вручну.

### Готовий → Оплачено

Після експорту в банк та виконання платежу — оновіть статус вручну або через інтеграцію з банком (наприклад, ПриватБанк API).

:::note
При використанні ПриватБанк API статус «Оплачено» може оновлюватися автоматично після виконання платежу.
:::

## Дії за статусами

| Статус | Доступні дії |
|--------|--------------|
| Новий | Переглянути, видалити |
| В обробці | Очікувати, скасувати |
| Готовий | Редагувати, експортувати, видалити |
| Оплачено | Переглянути, експортувати звіт |
| Відхилено | Переглянути причину, завантажити заново |
